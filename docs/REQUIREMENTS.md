# ZUTOMAYOカードデッキ作成＆管理アプリ 要件定義書 (REQUIREMENTS.md)

## 1. アプリケーション概要
ZUTOMAYOのカードゲームを楽しむユーザー向けに、カードの閲覧、検索、およびデッキ構築・管理を快適に行えるWebアプリケーション。スマホ・PC両対応のモダンなUIを提供する。データはすべて端末内で完結させることで、オフラインに近い感覚で超高速に動作するローカルファーストな設計とする。

## 2. ターゲットユーザー
- ZUTOMAYOカードゲームのプレイヤー
- カードコレクションを眺めたいファン
- 煩わしい登録なしですぐにデッキ構築を試したいユーザー

## 3. 主要機能要件
### 3.1. カード一覧・検索機能
- **全体見渡すモード（グリッド表示）**: 収録カード全体を一覧表示し、直感的にカードを探せるモード。
  - *パフォーマンス要件*: 画像が合計130MB以上（約440枚）存在するため、「Lazy Loading（遅延読み込み）」を実装し、画面スクロールに応じて画像を読み込むことで初期表示の重さを回避する。
- **拡大モード（詳細表示）**: 選択したカードの画像、名前、効果、属性などの詳細情報を大きく表示するモード。
  - *UX要件*: モーダルなどを使い、ユーザーがストレスなく詳細と一覧を切り替えられること。
- **検索・フィルタリング機能**:
  - キーワード検索（カード名、効果テキスト等）
  - シーズン（第1〜4弾など）、属性、種類などによるフィルタリング
  - ソート機能

### 3.2. デッキ作成・保存機能
- **デッキ構築**: カード一覧画面からカードを選択し、シームレスにデッキリストに追加できること。上限枚数等のバリデーションも考慮。
- **デッキ保存・管理 (ローカル保存)**:
  - 構築したデッキに任意の「デッキ名」をつけて、利用しているブラウザ端末（LocalStorage）に保存できる。アカウント登録等は不要。
  - 保存済みデッキの一覧表示。
  - 過去に作成したデッキの編集、削除機能。

## 4. デザイン・UI/UX要件
- **世界観**: ZUTOMAYOの世界観にマッチした、スタイリッシュでダークなデザイン。
- **レスポンシブデザイン**: スマートフォン、タブレット、PCに最適化。
- **インタラクション**: マイクロアニメーション（ホバー効果、画面遷移時のフェードなど）を取り入れ、操作していて心地よいUIを目指す。
